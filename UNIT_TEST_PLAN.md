# План Unit Тестування для проекту ReadAloud

Цей документ містить детальний список unit тестів для забезпечення повної перевірки основного функціоналу додатку. Тести розділені по доменах та сервісах.

## 1. Доменний рівень (Domain Layer)

### 1.1 TextProcessor (`src/domain/text_processor.py`)
Цей сервіс відповідає за очищення та підготовку тексту перед обробкою.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_process_text_removes_quotes` | Очищення тексту | Перевіряє, що одинарні (`'`) та подвійні (`"`) лапки видаляються з тексту. |
| `test_process_text_removes_special_markers` | Очищення тексту | Перевіряє видалення маркерів `***` та `======`. |
| `test_process_text_replaces_newlines` | Форматування | Перевіряє, що символи нового рядка `\n` замінюються на 4 пробіли. |
| `test_process_text_empty_input` | Обробка помилок | Перевіряє поведінку при порожньому рядку. |
| `test_process_text_none_input` | Обробка помилок | Перевіряє викидання `ValueError`, якщо вхідний текст `None`. |
| `test_process_text_complex_cleaning` | Комбінована очистка | Перевіряє текст, що містить одночасно лапки, маркери та нові рядки. |

### 1.2 TextChunker (`src/domain/text_chunker.py`)
Сервіс розбиття тексту на фрагменти (chunks) для озвучення.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_chunk_text_respects_boundaries` | Розбиття тексту | Перевіряє, що текст не розбивається посеред слова (використовуються межі пробілів/табуляції). |
| `test_chunk_text_sequencing` | Нумерація фрагментів | Перевіряє, що фрагменти (`AudioChunk`) мають послідовні номери, починаючи з 1. |
| `test_chunk_text_small_size` | Розбиття тексту | Перевіряє розбиття дуже короткого тексту, меншого за розмір чанка. |
| `test_chunk_text_exact_multiplier` | Розбиття тексту | Перевіряє розбиття, коли довжина тексту кратна розміру чанка. |
| `test_chunk_text_invalid_size` | Валідація | Перевіряє викидання `ValueError` при `chunk_size <= 0`. |
| `test_chunk_text_empty_string` | Валідація | Перевіряє викидання `ValueError` при порожньому вхідному тексті. |

### 1.3 AudioGenerator (`src/domain/audio_generator.py`)
Сервіс взаємодії з Edge TTS API. Потребує мокінгу `edge_tts.Communicate`.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_voice_mapping_coverage` | Конфігурація | Перевіряє наявність мапінгу для всіх підтримуваних мов (en, uk, de, ru) та обох статей. |
| `test_generate_audio_batch_calls_tts` | Інтеграція | Перевіряє, що метод викликає `edge_tts.Communicate` з правильними параметрами (текст, голос). |
| `test_generate_audio_batch_returns_paths` | Генерація файлів | Перевіряє повернення списку абсолютних шляхів до згенерованих mp3 файлів. |
| `test_generate_audio_batch_unsupported_lang` | Обробка помилок | Перевіряє викидання `ValueError` при запиті непідтримуваної мови. |
| `test_generate_audio_batch_failure_handling` | Відказостійкість | Перевіряє коректну обробку та журналювання помилок, якщо API повертає помилку. |
| `test_rate_limiting_semaphore` | Продуктивність | Перевіряє, що одночасно виконується не більше дозволеної кількості запитів (mock semaphore). |

### 1.4 AudioAssembler (`src/domain/audio_assembler.py`)
Сервіс склеювання аудіофайлів за допомогою ffmpeg. Потребує мокінгу `subprocess.Popen`.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_assemble_audio_creates_concat_file` | Команди ffmpeg | Перевіряє створення текстового списку файлів для конкатенації (формат `file 'path'`). |
| `test_assemble_audio_ffmpeg_command_structure` | Команди ffmpeg | Перевіряє правильність побудови команди `ffmpeg` (кодеки, фільтр темпу `atempo`). |
| `test_assemble_audio_speed_adjustment` | Зміна швидкості | Перевіряє додавання фільтра `atempo`, якщо швидкість відрізняється від 1.0. |
| `test_assemble_audio_progress_callback` | Зворотний зв'язок | Перевіряє виклик `callback` з відсотком прогресу при читанні stderr від ffmpeg. |
| `test_assemble_audio_empty_list` | Валідація | Перевіряє викидання `ValueError`, якщо список файлів порожній. |
| `test_assemble_audio_ffmpeg_error` | Обробка помилок | Перевіряє викидання виключення, якщо ffmpeg завершився з ненульовим кодом. |
| `test_stop_terminates_processes` | Керування ресурсами | Перевіряє, що метод `stop()` зупиняє всі активні процеси ffmpeg. |

## 2. Рівень інфраструктури (Infrastructure Layer)

### 2.1 FileManager (`src/infrastructure/file_manager.py`)
Робота з файловою системою. Потребує мокінгу `pathlib.Path` та `shutil`.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_create_timestamped_dir` | Організація файлів | Перевіряє створення папки з коректним форматом імені та датою. |
| `test_save_text_chunk_writes_content` | Запис даних | Перевіряє створення файлу `{number}.txt` з правильним вмістом. |
| `test_cleanup_temp_directories` | Очищення | Перевіряє видалення списку директорій (shutil.rmtree). |
| `test_ensure_directory_exists` | Створення папок | Перевіряє рекурсивне створення папок, якщо вони не існують. |
| `test_save_text_chunk_missing_dir` | Обробка помилок | Перевіряє помилку при спробі зберегти чанк у неіснуючу папку. |

### 2.2 ProgressTracker (`src/infrastructure/progress_tracker.py`)
Трекінг прогресу та оцінка часу.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_update_progress_percentage` | Розрахунки | Перевіряє правильність розрахунку відсотків (5/10 чанків = 50%). |
| `test_get_eta_string_valid_estimation` | Прогнозування | Перевіряє розрахунок приблизного часу завершення на основі середнього часу обробки чанка. |
| `test_get_eta_string_no_data` | Прогнозування | Перевіряє повернення "...", якщо обробка ще не розпочата або даних недостатньо. |
| `test_thread_safety` | Багатопотоковість | Перевіряє коректність лічильників при одночасному оновленні з декількох потоків. |
| `test_invalid_total_chunks` | Валідація | Перевіряє викидання `ValueError` при ініціалізації з `total_chunks <= 0`. |

## 3. Рівень застосунку (Application Layer)

### 3.1 AppController (`src/application/app_controller.py`)
Оркестрація процесу. Ці тести фокусуються на логіці викликів сервісів.

| Назва тесту | Функціонал, що перевіряється | Опис перевірки |
| :--- | :--- | :--- |
| `test_orchestration_flow` | Бізнес-логіка | Перевіряє послідовність: TextProcessor -> TextChunker -> AudioGenerator -> AudioAssembler. |
| `test_pause_resume_logic` | Керування завданнями | Перевіряє зупинку та відновлення черги генерації. |
| `test_task_deletion_resource_cleanup` | Керування ресурсами | Перевіряє, що при видаленні завдання зупиняються генератори та видаляються тимчасові файли. |
| `test_concurrent_tasks_management` | Багатозадачність | Перевіряє коректність роботи контролера при декількох активних завданнях у черзі. |
